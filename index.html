ModeSyntax ExampleBytesEffective Address / ValueNotes
ImpliedCLC / RTS / INX1No operand. Operates on register(s) only.Fastest — no memory fetch beyond opcode itself.
AccumulatorASL A / LSR A / ROL A1Operates directly on the A register.Some assemblers omit the "A" — context-dependent.
ImmediateLDA #$42 / AND #%111100002Value is the literal byte following the opcode.Fast (2 cycles). # prefix. Can use decimal, hex, or binary.
Zero PageLDA $C0 / STA $FB2Address is $00xx — high byte assumed $00.3 cycles. ZP is fastest RAM access. Reserve $00-$FF for critical vars.
Zero Page, XLDA $C0,X / STA $80,X2Address = $00 + lowbyte + X (wraps within page 0)4 cycles. Ideal for small indexed tables in ZP. X wraps: $FE+3 = $01.
Zero Page, YLDX $20,Y / STX $30,Y2Address = $00 + lowbyte + Y (wraps within page 0)Only LDX and STX use ZP,Y. 4 cycles.
AbsoluteLDA $D020 / STA $60003Full 16-bit address in operand (lo-byte first).4 cycles. Standard HW register access mode.
Absolute, XSTA $D000,X / LDA $6000,X3Address = full 16-bit base + X4-5 cycles (+1 if page crossed). Loop through tables/screen RAM.
Absolute, YLDA $6000,Y / STA $C000,Y3Address = full 16-bit base + Y4-5 cycles (+1 if page crossed). Like abs,X but with Y.
IndirectJMP ($0300)3Reads 16-bit pointer from given address, jumps there.JMP only. NMOS bug: JMP ($xxFF) reads high byte from $xx00 not $xx+1,00.
(Indirect, X)LDA ($20,X) / STA ($40,X)2ZP address = lowbyte+X; read 16-bit pointer from there.6 cycles. X selects which pointer in a ZP table. X usually 0 or small.
(Indirect), YLDA ($20),Y / STA ($FB),Y2Read 16-bit pointer from ZP address; add Y for final address.5-6 cycles. Most flexible. Pointer in ZP, Y is row/column offset.

Addressing Mode Decision Guide

GoalBest ModeWhy
Read a hardware register (VIC/SID/CIA)AbsoluteHW regs are all above $D000, need full 16-bit address
Fast frequently-accessed variableZero Page2-byte instruction, 3 cycles, page 0 only
Loop through a 256-byte tableAbsolute,X or ZP,XIncrement X each iteration, no pointer update needed
Copy data via a pointer (variable dest)(Indirect),YStore pointer in ZP, Y steps through bytes
Select one of several sub-tables(Indirect,X)X picks the ZP pointer, pointer holds the table base
Conditional loop / early exitRelative (branch)Only mode for branches; keep loops <128 bytes or use JMP
Sprite X positions ($D000,$D002,...)Absolute,X (X=0,2,4...)Step X by 2 to hit each sprite register pair
Screen RAM fill (1000 bytes)Absolute,X / Absolute,YX or Y as counter; need outer loop for >256 bytes

6502 / 6510 Architecture

A — Accumulator
████████ (8-bit)

Primary arithmetic and logic register. All ALU operations use A. Only register that can do ADC/SBC. Load with LDA, store with STA, transfer to X/Y with TAX/TAY.

X — Index Register
████████ (8-bit)

Index register for abs,X and zp,X addressing. Loop counter. Only X can set the Stack Pointer via TXS. Compare with CPX. Transfer A↔X via TAX/TXA.

Y — Index Register
████████ (8-bit)

Index register for abs,Y and (ind),Y addressing. The latter is the most powerful 6502 indirect mode. Compare with CPY. Transfer A↔Y via TAY/TYA.

PC — Program Counter
████████████████ (16-bit)

Points to next instruction byte. Modified by JMP, JSR, RTS, RTI, BRK, and all branch instructions. Cannot be directly read or written — only indirectly via stack.

SP — Stack Pointer
████████ (8-bit, page 1)

Always points within $0100-$01FF. Decrements on push, increments on pull. Initialized by LDX #$FF / TXS at startup. Read via TSX. PHA/PHP push; PLA/PLP pull.
