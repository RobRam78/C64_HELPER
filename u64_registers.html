U64 REGISTER REFERENCE

U64
C64 Dev Machine · Polytricity Ltd
Two distinct layers of I/O registers are accessible from running 6502 code. The standard C64 chip registers ($D000–$DDFF) are fully emulated by the U64. The U64-specific extension registers sit in the same address space — $D030/$D031 for CPU turbo control, $DF00–$DF0A for REU DMA, and $DF1B–$DF1F for the Ultimate Command Interface. All addresses assume CHAREN=1 (I/O visible, not character ROM).

VIC-II VIDEO CHIP

$D000–$D02E
ADDRESSREGISTERNOTES
$D000/$D001Sprite 0 X / YScreen position. X is 8-bit here; bit in $D010 provides 9th bit.
$D002–$D00DSprites 1–6 X / YSame layout as sprite 0, pairs of X/Y bytes.
$D00E/$D00FSprite 7 X / YFinal sprite position pair.
$D010Sprites X MSBsR/W One bit per sprite. Set bit N to extend sprite N X coordinate beyond pixel 255. Used in MACRO_ANIM for 9th-bit X handling.
$D011VIC Control 1R/W Bit 7 = raster IRQ bit 8. Bit 6 = ECM. Bit 5 = BMM (bitmap mode). Bit 4 = DEN (display enable). Bit 3 = RSEL (25/24 rows). Bits 0–2 = YSCROLL.
$D012Raster CounterR/W Read = current raster line (bits 0–7). Write = IRQ trigger line (combine with $D011 bit 7 for lines > 255).
$D013/$D014Light Pen X / YR Latched pen position on trigger.
$D015Sprite EnableR/W Bit per sprite. Set to show, clear to hide.
$D016VIC Control 2R/W Bit 4 = MCM (multicolour text). Bit 3 = CSEL (40/38 cols). Bits 0–2 = XSCROLL. Key for MACRO_SCROLL fine-scroll logic.
$D017Sprite Y ExpansionR/W Doubles sprite height when bit set.
$D018Memory PointersR/W Bits 4–7 = screen RAM offset within VIC bank (×$0400). Bits 1–3 = charset/bitmap base (×$0800). Critical for VIC bank switching in MACRO_SPR.
$D019IRQ StatusR/W Bit 0 = raster IRQ pending. Bit 1 = sprite-bg collision. Bit 2 = sprite-sprite collision. Bit 3 = light pen. Write 1 to acknowledge.
$D01AIRQ Enable MaskR/W Same bit layout as $D019. Set bit to enable that IRQ source.
$D01BSprite PriorityR/W 0 = sprite in front of background, 1 = behind.
$D01CSprite MulticolourR/W Enable multicolour mode per sprite. Shared colours from $D025/$D026.
$D01DSprite X ExpansionR/W Doubles sprite width when bit set.
$D01ESprite–Sprite CollisionR Bits set for colliding sprite pairs. Cleared on read.
$D01FSprite–BG CollisionR Set when sprite overlaps non-zero background pixel. Used in MACRO_COLL. Cleared on read.
$D020Border ColourR/W Bits 0–3 only. 16 colours.
$D021Background Colour 0R/W Main background colour for text and bitmap modes.
$D022–$D024Background Colours 1–3R/W Extra background colours for ECM text mode.
$D025/$D026Sprite Shared MC 0/1R/W Two shared multicolour values used by all MC sprites.
$D027–$D02ESprite 0–7 ColourR/W Individual colour for each sprite (bits 0–3).
STANDARD C64

SID SOUND CHIP

$D400–$D41C
U64 NOTE
The U64 can emulate up to 4 SID chips at configurable addresses (e.g. $D400, $D420, $D440, $D480 — set in the hardware menu). Second SID address is configurable. Your MACRO_SID node should target the address matching the U64's configured primary SID.
ADDRESSREGISTERNOTES
$D400/$D401V1 Frequency Lo/Hi16-bit frequency. freq = note_hz × 16777216 / clock (PAL clock ≈ 985248 Hz).
$D402/$D403V1 Pulse Width Lo/Hi12-bit pulse width (bits 0–11 of $D402/D403). 50% duty = $0800.
$D404V1 ControlBit 0=Gate, 1=Sync, 2=Ring, 3=Test, 4=Tri, 5=Saw, 6=Pulse, 7=Noise.
$D405V1 Attack/DecayHigh nibble=Attack (0–15), Low nibble=Decay (0–15).
$D406V1 Sustain/ReleaseHigh nibble=Sustain level, Low nibble=Release rate.
$D407–$D40DVoice 2Same register layout as Voice 1.
$D40E–$D414Voice 3Same register layout as Voice 1.
$D415/$D416Filter Cutoff Lo/Hi11-bit cutoff. Lo = bits 0–2, Hi = bits 3–10.
$D417Filter Resonance/RouteHigh nibble=resonance. Bits 0–2=voice routing to filter. Bit 3=ext input.
$D418Volume / Filter ModeBits 0–3=master volume. Bits 4–6=filter type (LP/BP/HP). Bit 7=V3 off.
$D419/$D41APaddle X / YR Paddle pot values (0–255). Requires CIA port setup.
$D41BV3 Oscillator OutputR Live waveform output of Voice 3. Useful for pseudo-random number generation.
$D41CV3 Envelope OutputR Live envelope value of Voice 3 (0–255).
STANDARD C64

CIA #1 — KEYBOARD / JOYSTICK / IRQ

$DC00–$DC0F
ADDRESSREGISTERNOTES
$DC00Port AR/W Keyboard column select (write) / Joystick Port 2 (read bits 0–4).
$DC01Port BR Keyboard row read / Joystick Port 1 (bits 0–4). Used in MACRO_JOY for port 1.
$DC02/$DC03Data Direction A/BR/W 1 = output, 0 = input per bit.
$DC04/$DC05Timer A Lo/HiR/W 16-bit countdown timer. Used for CIA-based IRQ timing in MACRO_SID.
$DC06/$DC07Timer B Lo/HiR/W Second independent timer. Can count Timer A underflows.
$DC08–$DC0BTOD ClockR/W Tenths, seconds, minutes, hours (BCD). Reading $DC08 latches all four.
$DC0CSerial Data RegisterR/W Shift register for serial communication.
$DC0DInterrupt ControlR/W Read = IRQ status flags. Write bit 7=1 to set mask, =0 to clear. Bit 0=Timer A, 1=Timer B, 4=FLAG.
$DC0EControl Register AR/W Bit 0=start Timer A. Bit 3=one-shot. Bit 4=load latch. Bit 5=timer mode.
$DC0FControl Register BR/W Same as $DC0E but for Timer B. Bit 5–6 selects count source.
STANDARD C64

CIA #2 — VIC BANK / SERIAL BUS / NMI

$DD00–$DD0F
DEV MACHINE NOTE
$DD00 is the most important CIA2 register for VIC bank switching. Bits 0–1 are inverted: writing %xxxxxx11 selects bank 0 ($0000), %xxxxxx10 = bank 1 ($4000), %xxxxxx01 = bank 2 ($8000), %xxxxxx00 = bank 3 ($C000). Your MACRO_SPR already reads $DD00 and $D018 live to resolve sprite pointers at runtime.
ADDRESSREGISTERNOTES
$DD00Port AR/W Bits 0–1 = VIC bank select (inverted). Bits 3–7 = serial bus lines (CLK, DATA, ATN, etc.).
$DD01Port BR/W User port I/O.
$DD02/$DD03Data Direction A/BR/W Bit direction for ports A and B.
$DD04–$DD0BTimers / TODSame layout as CIA1 timer and TOD registers.
$DD0CSerial Data RegisterR/W Serial shift register.
$DD0DNMI ControlR/W Like $DC0D but triggers NMI instead of IRQ. Bit 0=Timer A, 1=Timer B, 4=FLAG.
$DD0E/$DD0FControl Registers A/BR/W Same layout as CIA1 control registers.
U64 ONLY

TURBO / CPU SPEED CONTROL

$D030, $D031, $D07A/$D07B, $D0BC
PREREQUISITE
These registers are only active when Turbo Mode in the U64 menu is set to "U64 Turbo Registers" or "Turbo Enable Bit". In any other mode they read back $FF. Detection: LDA $D031 : CMP #$FF : BEQ not_u64.
ADDRESSREGISTERDETAIL
$D031 U64 Turbo Control R/W Bits 0–3 = CPU speed index (0–15, see table below).
Bit 7 = Badline timing: 0 = enabled (C64-accurate), 1 = disabled (more CPU cycles).
Active under U64 Turbo Registers or Turbo Enable Bit menu mode.
$D030 Turbo Enable Bit R/W Active only under Turbo Enable Bit mode.
Write bit 0: 0 = force 1MHz + badlines, 1 = use menu speed settings.
Read bit 0: 0 = turbo off, 1 = turbo on.
$D07A SuperCPU Normal Speed W SuperCPU-compatible write-only. Forces CPU to normal (1MHz) speed.
$D07B SuperCPU Turbo Speed W SuperCPU-compatible write-only. Forces CPU to turbo (20MHz) speed.
$D0BC SuperCPU Detect R Read-only detection register. Requires separate enable in the U64 menu.
IMPORTANT CAVEAT
Disabling badlines via bit 7 of $D031 does not affect the BA signal on the cartridge port. Accesses to external devices — SID socket chips, cartridge ROM — still cause badline stalls. The setting only applies to internal memory accesses.

RAM EXPANSION UNIT (REU) DMA

$DF00–$DF0A
OVERVIEW
The U64's built-in REU is compatible with the original Commodore 1700/1764/1750 register layout, extended with extra address bits for up to 32MB of REU RAM. A single DMA command at $DF01 transfers data between REU and C64 RAM invisibly — the CPU does not execute during the transfer. Ideal for streaming large assets that do not fit in the 64K C64 address space.
ADDRESSREGISTERNOTES
$DF00Status RegisterR Bit 7=transfer done. Bit 6=end-of-block. Bit 5=fault. Bit 4=size (1=256 bytes). Cleared on read.
$DF01Command RegisterW Bit 7=execute now. Bit 4=FF00 trigger. Bits 0–1=transfer type: 00=C64→REU, 01=REU→C64, 10=swap, 11=verify.
$DF02/$DF03C64 Base AddressR/W 16-bit C64 RAM address (lo/hi). Auto-increments after transfer unless bit 7 of $DF0A set.
$DF04/$DF05/$DF06REU Base AddressR/W 24-bit REU address (lo/mid/hi). $DF06 high bits extend into multiple megabytes on U64.
$DF07/$DF08Transfer LengthR/W 16-bit byte count (lo/hi). Value 0 = 65536 bytes.
$DF09Interrupt MaskR/W Bit 7=enable interrupt. Bit 6=on end-of-block. Bit 5=on verify fault.
$DF0AAddress ControlR/W Bit 7=fix C64 address. Bit 6=fix REU address. When fixed, address does not auto-increment during transfer.
PATTERN — STREAM REU → C64
STA $DF03 / STA $DF02 — set C64 destination.
STA $DF06 / STA $DF05 / STA $DF04 — set REU source address.
STA $DF08 / STA $DF07 — set byte count.
LDA #$91 : STA $DF01 — execute REU→C64 transfer immediately (bit 7=execute, bits 0–1=01).
U64 ONLY

ULTIMATE COMMAND INTERFACE (UCI)

$DF1B–$DF1F
PREREQUISITE
The UCI registers must be enabled in the U64 menu under Command Interface settings. They occupy the last 5 bytes of the REU address range — enabling UCI masks off those REU registers.
ADDRESSDIRREGISTERNOTES
$DF1BRSoftwareIEC Bus IDDevice ID of the Software IEC virtual drive.
$DF1CWControl RegisterBit 0=PUSH_CMD (fire queued command). Bit 1=DATA_ACC (acknowledge response). Bit 2=ABORT. Bit 3=CLR_ERR. Bit 5=IRQ enable. Bit 6=TRIGGER. Bit 7=DMA.
$DF1CRStatus RegisterBit 7=data available. Bit 6=status available. Bits 4–5=protocol state (00=Idle, 01=CmdBusy, 10=DataLast, 11=DataMore). Bit 2=error. Bit 0=cmd busy.
$DF1DWCommand Data RegisterWrite command bytes here before firing PUSH_CMD. First byte is target ID.
$DF1DRIdentification RegisterReads $C9 at idle. Reads $49 when IRQ is active (bit 7 cleared).
$DF1ERResponse Data QueueRead response bytes here when bit 7 of $DF1C status is set.
$DF1FRStatus Data QueueRead command result/status text here when bit 6 of $DF1C status is set.

CONTROL TARGET ($04) COMMANDS

COMMAND BYTESACTION
$04 $01Identify — returns version string e.g. "CONTROL TARGET V1.1"
$04 $05Freeze — triggers hardware freeze, equivalent to pressing the button
$04 $06Reboot — full C64 reset. No response expected (UCI resets too)
$04 $0F <filename>Save RAM — dumps all 64K C64 RAM to file (default: /temp/c64_memory.bin)
$04 $29Get Drive Info — returns count + [type, IEC address, power state] per drive
$04 $30 / $31Enable / Disable emulated Drive A
$04 $32 / $33Enable / Disable emulated Drive B
$04 $08 / $09 <file>Load / Save REU image to/from SD card storage

POTENTIAL MACRO NODES

★ MACRO_TURBO
Wraps $D031/$D030. Emits a U64 detection probe at init, sets speed index for compute-heavy sections, restores 1MHz before raster IRQ. "Burst compute" pattern.
$D031 · $D030 · $D07A · $D07B
★ MACRO_REU
REU DMA for streaming large data. Double-buffer bitmap frames, pre-build scroll buffers, store multiple music tracks. Invisible to CPU — transfers happen in hardware.
$DF00 · $DF01 · $DF02–$DF0A
MACRO_U64_DETECT
Simple probe: LDA $D031 / CMP #$FF / BEQ not_u64. Sets a zero-page flag. Other nodes branch on this to conditionally enable U64 features.
$D031
MACRO_UCI
Lets running demo code trigger system actions: freeze to menu on completion, reboot, save RAM for debug dumps, query/toggle emulated drives.
$DF1B · $DF1C · $DF1D · $DF1E · $DF1F
RECOMMENDED PRIORITY
MACRO_TURBO and MACRO_REU are the highest-value additions. Turbo enables fast setup code and complex per-frame computation. REU enables asset streaming beyond the 64K address limit — multiple bitmap pages, multi-track SID data, large map data — without touching C64 RAM layout at all.
